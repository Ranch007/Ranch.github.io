超详细的wsl2教程：windows上的linux子系统

2024-11-02
2024-11-02T22:47
827 字
阅读时长: 2 分钟

前置条件

1.1 CPU 虚拟化

  • 修改BIOS设置（VMX/AMD-v）

1.2 windows 功能

  • 适用于Linux的windows子系统
  • 虚拟机平台

安装

2.1 默认 ubuntu 发行版

管理员权限打开 cmd

1

wsl --install kali-linux --web-download

2.2 查看在线发行版 

1

wsl --list --online

2.3 查看安装的子系统 

1

wsl --list -v

如果 windows下 sublinux 安装出现“不能访问网络位置+有关网络排除故障的信息”

1
2
3
4

netsh winsock reset
netsh int ip reset all
netsh winhttp reset proxy
ipconfig /flushdns

使用

3.1 切换默认子系统 

1

wsl --set-default xxx

3.2 启动指定的子系统 

1

wsl -d kali-linux

3.3 卸载 

1

wsl --unregister kali-linux

3.4 备份 

1
2
3
4
5

# 导出
wsl --export kali-linux kali.tar

# 导入
wsl --import kali2 D:/wsl C:\Users\admin\Desktop

kali 打开桌面窗口

4.1 wsl2 黑科技

4.1.1 WSLg

允许 Linux 里面带有UI的应用程序，直接以 Windows 窗口的形式打开

4.1.2 显卡直通 

1
2

# 查看显卡
nvidia-smi

4.1.3 镜像网络模式

在 C:\Users\xxx 下面创建 .wslconfig 文件

1
2

[wsl2]
networkingMode = mirrored

然后保存，使用 wsl --shutdown 关闭虚拟机，等 8 秒。

4.2 kali 安装 Win-Kex

参考官方文档：【Win-Kex为Kali Linux提供GUI桌面体验】

win-kex支持以下三种模式

4.2.1 窗口模式

要在支持声音的 Window 模式下启动 Win-KeX，请运行以下任一命令：

  • Kali WSL 内部：kex --win -s
  • 在 Windows 的命令提示符上：wsl -d kali-linux kex --win -s

有关更多信息，请参阅【 Win-KeX 窗口模式使用文档】。

4.2.2 增强的会话模式

要在具有声音支持和 ARM 解决方法的增强会话模式下启动 Win-KeX

请运行以下任一命令：

  • Kali WSL 内部：kex --esm --ip -s
  • 在 Windows 的命令提示符上：wsl -d kali-linux kex --esm --ip -s

有关更多信息，请参阅【 Win-KeX 增强型会话模式使用文档】。

4.2.3 无缝模式

使用无缝模式（win-kex SL），请在WSL设置里，将网络模式更改为NAT

进入kali系统，修改/etc/resolv.conf文件里的nameserver，其IP地址与windows中vEthernet (WSL (Hyper-V firewall))IP地址一致。

要在具有声音支持的无缝模式下启动 Win-KeX，请运行，运行以下任一：

  • Kali WSL 内部：kex --sl -s
  • 在 Windows 的命令提示符上：wsl -d kali-linux kex --sl -s

有关更多信息，请参阅【 Win-KeX SL 使用文档】。
